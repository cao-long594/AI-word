## 三、核心技术实现

### 1. API Key 与本地存储方案
根据作业要求，API Key 不直接写入源码，也不提交到仓库，而是使用浏览器本地存储进行手动保存。当前实现中使用固定键名 `LINGXI_API_KEY` 保存密钥，页面运行时从 `localStorage` 读取该值并完成接口鉴权。除了 API Key 之外，主题状态和聊天记录也可以通过本地存储进行持久化，这种实现方式能有效避免把密钥暴露在公开代码中。

### 2. AI 对话与流式输出

项目通过 Fetch API 调用阿里云百炼兼容接口，请求体中传入模型名称、消息数组以及流式输出参数。响应阶段采用 `ReadableStream` 持续读取服务端返回数据，再将增量文本逐步拼接到当前消息气泡中，从而实现类似“打字机”的逐字输出效果。用户在等待 AI 输出时可以点击停止按钮，通过 `AbortController` 中断请求，避免长时间生成影响操作体验。

### 3. Markdown 渲染与代码高亮

AI 返回的文本在展示前会先进行 Markdown 解析，再转换为 HTML 插入消息区域，以保证标题、表格、无序列表、引用、代码块等内容可以正确渲染。针对代码块，项目额外接入语法高亮库，增强代码阅读效果；同时在代码区域右上角提供复制按钮，方便用户快速复制示例代码或命令行内容。

### 4. 主题切换与本地持久化

页面提供深色模式和浅色模式切换能力，主要通过给页面根节点切换主题类名，并结合 CSS 变量统一管理颜色体系实现。刷新页面后，程序会从浏览器 `localStorage` 中读取主题值并恢复用户上次选择，从而保证主题切换不是一次性效果，而是具有持久化能力的界面设置。


## 四、项目亮点与补充说明

### 1. 增强的多模态交互体验

本项目在满足课程基础要求的同时，也尽量兼顾了界面完整性和实际使用体验。例如，流式响应和打字机动画能够让用户明显感知 AI 正在生成内容，而不是长时间等待后一次性出现结果；代码高亮以及复制按钮则更适合技术类问答场景。

特别地，**图片上传与预览功能**得到了深度优化：
- 提供缩略图预览，用户可点击缩略图放大查看
- 在图片预览模态框中实现 **Ctrl/Cmd + 滚轮缩放** 功能，放大倍数范围从 0.5 倍到 5 倍，满足不同分辨率和尺寸图片的查看需求
- 关键优化处理：竖长图片预览时高度自动限制在 `calc(100vh - 100px)` 以内，避免超出窗口边界
- 智能阻止 Ctrl + 滚轮缩放页面，仅允许缩放图片内容本身，优化了交互冲突

### 2. 用户界面优化与细节设计

为了提升聊天体验，项目还实现了多项 UI/UX 细节：

- **自动滚动管理**：当用户在底部时，新消息到达或 AI 响应生成时自动平滑滚动到底部；当用户向上翻阅历史消息时，自动滚动功能暂停，避免打断用户阅读
- **返回底部按钮**：当用户向上滚动离开视频底部时，会显示"返回底部"浮窗，点击后平滑回到最新消息
- **Toast 通知系统**：重要操作反馈（如复制成功、清除对话等）通过屏幕底部 Toast 提示，3 秒后自动消失，不打断流程
- **文本框自适应**：输入框高度随输入文本行数自动调整，单行时为常规高度，多行文本时逐步扩展高度但不超过视区的一半

### 3. 代码组织与可维护性

项目采用了良好的代码组织实践，提升了长期维护性和扩展性：

- **常量配置集中管理**：将 API 端点、模型名称、localStorage 键名等常量统一定义在文件顶部，便于后续配置修改而无需遍历整个代码库
- **统一的 Markdown 渲染管线**：AI 响应内容渲染遵循固定流程（解析 → 高亮 → 集成），通过专一函数 `renderMarkdownToBubble()` 实现，避免了代码散落和逻辑重复
- **事件监听生命周期管理**：图片预览模态框内的滚轮缩放监听在打开时动态添加，关闭时科学清理，防止内存泄漏

### 4. 边界情况与容错处理

项目在多个方面考虑了边界和容错：

- API Key 缺失或无效时，在控制台提示完整的设置步骤，便于用户快速定位问题
- 代码块语言标识识别失败时降级显示为 "code"，保证页面不出错
- 所有 localStorage 操作都包围在 try-catch 中，兼容隐私浏览模式等场景
- Markdown 解析错误时记录到控制台而非中断页面，保证聊天流程的连贯性
