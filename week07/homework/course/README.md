## 三、核心技术实现

- 路由与权限：`client/src/router` 使用 `BrowserRouter` 管理页面路由，通过 `RequireAuth` 判断本地登录状态，未登录时自动跳转到登录页，已登录访问登录页时跳转到工作台。
- 请求封装：`client/src/api/request.ts` 基于 axios 创建统一实例，设置 `baseURL: '/api'`，请求前自动携带 Token，响应后统一解包 `{ code, msg, data }`，并集中处理接口错误和登录失效。
- 页面状态管理：课程和学生页面使用 React state 保存列表、加载状态、分页、筛选条件和弹窗状态；查询表单负责收集筛选条件，表格负责展示结果，弹窗表单负责新增和编辑。
- 表格操作交互：课程和学生列表中的编辑、删除、发布、下架、启用、停用等操作都通过按钮触发对应 API，请求成功后重新加载列表，保证页面数据和后端保持一致。
- 图表渲染：工作台页面拿到统计数据后交给 Recharts 渲染柱状图、折线图等图表，页面只负责数据展示和空状态处理，图表配置集中在组件内部。
- Markdown 渲染：学习总结页使用 `react-markdown` 和 `remark-gfm` 渲染 Markdown 内容，并重写 `code/pre` 渲染逻辑，让代码块支持语言标识、基础语法高亮、一键复制和复制成功对钩反馈。
- 样式组织：全局样式集中在 `client/src/index.css`，统一管理后台布局、菜单、卡片、表格、按钮、表单、Markdown 内容和响应式适配，保证不同页面视觉风格一致。
- 开发代理：`client/vite.config.ts` 配置 Vite proxy，把开发环境下的 `/api` 请求转发到 `http://localhost:3000`，前端代码在开发和生产环境都可以使用同一套 `/api` 请求路径。
